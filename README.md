Проектирование и разработка мобильного приложения для анализа качества почвы
Роль в проекте: Бизнес-системный аналитик / Lead Developer.

Описание проекта
Информационная система нового поколения для агропромышленного комплекса, направленная на автоматизацию сбора, хранения и интеллектуального анализа показателей плодородия почвы. Проект решает проблему фрагментарности существующих решений и их зависимости от наличия интернет-соединения на удаленных сельхозугодьях.

Ключевые функции (Бизнес-ценность)
Автономность (Offline-first): Полная функциональность и проведение расчетов без доступа к сети.
Интеллектуальный расчет: Автоматическое формирование научно обоснованных рекомендаций по внесению удобрений на основе расчетно-балансового метода.
Экологическая оптимизация: Снижение избыточного внесения химикатов и минимизация негативного воздействия на экосистемы.
Управление ролями: Разграничение прав доступа для Агронома, Руководителя и Администратора.
Что я сделала в рамках проекта (Аналитика и разработка)
1. Системный анализ и проектирование требований
Провела комплексный анализ предметной области и выявила ограничения существующих систем (фрагментарность, зависимость от внешних источников).
Разработала функциональные и нефункциональные требования к системе.
Спроектировала модель пользовательского взаимодействия и описала основные сценарии использования (Use Case диаграммы в UML).
2. Архитектурное проектирование
Разработала инфологическую модель данных (ER-диаграмму) для обеспечения целостности агрохимических показателей.
Спроектировала трехслойную архитектуру приложения (Data, Domain, Presentation) на основе принципов Clean Architecture.
Обосновала выбор технологического стека: Flutter (Dart) для кроссплатформенности и SQLite для надежного локального хранения данных.
3. Математическое моделирование
Интегрировала в ядро системы (Domain Layer) математическую модель расчетно-балансового метода для вычисления физической дозы удобрений.
Реализовала алгоритм автоматического подбора корректирующих коэффициентов в зависимости от типа почвы (чернозем, подзолистая и др.).
4. Реализация и верификация
Разработала программный прототип мобильного приложения.
Провела верификацию алгоритмов: контрольное сравнение автоматизированных расчетов с ручными показало 100% точность при скорости обработки данных в 0.2 секунды.
Настроила механизмы валидации данных (например, ограничение шкалы pH и блокировка отрицательных значений) для защиты от ошибок ввода.
Стек технологий
Анализ и проектирование: UML, ER-диаграммы, BPMN (сценарии использования).
Mobile Development: Flutter, Dart.
Database: SQLite (sqflite).
Architecture: Clean Architecture, BLoC/Provider (State Management).
